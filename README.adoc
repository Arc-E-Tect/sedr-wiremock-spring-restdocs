== Introduction

This repository is a standalone spin-off from the book _Software Engineering Done Right_ (SEDR).
It highlights a specific practice from the book that deserves its own dedicated focus: automatically generating WireMock stubs from Spring REST Docs contract tests, and using those stubs to drive Angular micro-frontend development without requiring the full backend to be running.

The example application is called *IFF — Identification: Friend or Family*.
It is split into two top-level projects that mirror a real-world micro-frontend + microservices setup:

`red-thread/`::
A Gradle multi-project build containing five Spring Boot microservices (Java 21):
`auth-server`, `user-account`, `relationship`, `social-network`, and `system-admin`.
Each service uses Spring REST Docs to write contract tests that simultaneously validate the API and emit WireMock stub mappings as a build artefact.

`blue-needle/`::
An Angular 21 micro-frontend with one lazy-loaded route per IFF microservice.
It can run against the generated WireMock stubs (no IFF build needed) or against the full running application stack.
A Cucumber + Playwright BDD smoke suite validates both modes.

The full explanation of how stub generation works, why it matters, and how to run the project end-to-end is in link:TUTORIAL.adoc[TUTORIAL.adoc].

== Quick start

=== Red Thread (backend / stub generation)

.Prerequisites
* Java 21 (tested with https://docs.aws.amazon.com/corretto/latest/corretto-21-ug/downloads-list.html[Amazon Corretto 21])
* Docker

[source,console]
----
# Build all microservices and generate WireMock stubs
cd red-thread
./gradlew build

# Generate stubs only (faster, skips unit/integration tests)
./gradlew :system-admin:docGen :user-account:docGen :auth-server:docGen \
          :relationship:docGen :social-network:docGen --no-daemon
----

=== Blue Needle (frontend against WireMock stubs)

.Prerequisites
* Node.js 22 or 24
* Docker

[source,console]
----
cd blue-needle
npm install

# Start WireMock containers with normalised stubs
npm run wiremock:up

# Start the Angular dev server
npm start
----

Open http://localhost:4200 in a browser.

See link:blue-needle/README.md[blue-needle/README.md] and link:red-thread/README.md[red-thread/README.md] for detailed instructions on each project.
